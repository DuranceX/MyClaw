# 输入框增强：命令补全、多行输入与历史记录

对聊天输入框进行了三项增强：斜杠命令系统（含浮层补全）、Shift+Enter 多行输入、上下键历史记录切换。

当前相关代码位于：

- `packages/web/app/chat.tsx` — 所有输入框逻辑
- `packages/web/next.config.ts` — 命令相关 API 的 rewrite 规则
- `packages/server/app/main.py` — 命令对应的后端接口

---

## 斜杠命令系统

### 背景与动机

原来的输入框只能发消息给 AI，没有办法执行本地操作（清屏、查看技能列表、切换模型等）。参考 Claude Code 和 better-claw 的实现，加入斜杠命令支持。

### 设计思路

命令在**前端拦截**，不走 `sendMessage()`，而是直接执行或调用后端专用接口，结果以 assistant 消息形式插入对话列表。这样命令结果和 AI 回复在视觉上统一，但不会污染发给模型的对话历史。

```
用户输入 /usage
    ↓
前端 handleCommand() 拦截
    ↓
fetch('/api/usage') → Python 后端
    ↓
结果格式化为 Markdown，insertSystemMessage() 插入对话
```

### 已实现的命令

| 命令 | 说明 | 处理位置 |
|------|------|------|
| `/clear` | 清空当前对话 | 纯前端 |
| `/skills` | 列出所有可用技能 | 调 `/api/skills` |
| `/models [provider]` | 列出 provider 或其下的模型 | 调 `/api/models` |
| `/model <provider> <id>` | 切换模型，写回 config.yaml | 调 `/api/model` |
| `/usage` | 查看 token 用量统计 | 调 `/api/usage` |

### 命令补全浮层

输入 `/` 时，根据已输入内容过滤 `SLASH_COMMANDS` 数组，显示匹配的候选列表。

浮层用 `absolute bottom-full` 定位，浮在输入框上方，不占用布局空间：

```tsx
const suggestions = input.startsWith('/')
  ? SLASH_COMMANDS.filter(c => c.name.startsWith(input.slice(1).split(' ')[0]))
  : [];
const showSuggestions = suggestions.length > 0 && !input.slice(1).includes(' ');
```

键盘交互：
- `↑` / `↓`：移动选中项（高亮蓝色背景）
- `Tab` 或 `Enter`（有选中项时）：填充命令名
- 点击：填充并关闭浮层
- `Escape`：关闭浮层并清空输入

### API 路由问题

前端命令直接 `fetch('/api/skills')` 等路径，本地开发时 Next.js 没有这些路由会 404。服务器上 Nginx 的 `location /api/` 会直接转发到 Python，所以服务器没问题。

解决方案：在 `next.config.ts` 加 rewrites，本地开发时也透传到 Python：

```ts
async rewrites() {
  return [
    { source: '/api/skills',  destination: `${BACKEND}/api/skills` },
    { source: '/api/models',  destination: `${BACKEND}/api/models` },
    { source: '/api/model',   destination: `${BACKEND}/api/model` },
    { source: '/api/usage',   destination: `${BACKEND}/api/usage` },
  ];
}
```

## 多行输入（Shift+Enter 换行）

### 改动

把 `<input>` 换成 `<textarea rows={1}`，加 `resize-none`，通过 `useEffect` 监听 `input` 变化自动撑高：

```tsx
useEffect(() => {
  const el = textareaRef.current;
  if (!el) return;
  el.style.height = 'auto';
  el.style.height = `${el.scrollHeight}px`;
}, [input]);
```

最大高度限制为 `8rem`（约 4 行），超出后出现滚动条。

键盘行为：
- `Enter`（无 Shift）：提交消息，同时重置高度（`style.height = 'auto'`）
- `Shift+Enter`：不拦截，让浏览器默认插入换行

## 输入历史记录（上下键切换）

### 方案选择

讨论过两个方案：

**方案 A：从聊天记录里取用户消息**
- 缺点：语义不同（聊天记录不含命令历史）；多行长消息体验差；依赖异步加载

**方案 B：独立维护，持久化到 localStorage**
- 优点：立即可用，包含命令历史，刷新后保留

**最终选择方案 B**，50 条上限，自动去重（同样内容不重复存）。

### 实现

```tsx
const [inputHistory, setInputHistory] = useState<string[]>(() => {
  try {
    return JSON.parse(localStorage.getItem('chat-input-history') ?? '[]');
  } catch { return []; }
});
```

发送时写入：

```tsx
setInputHistory(prev => {
  const next = [text, ...prev.filter(h => h !== text).slice(0, 48)];
  localStorage.setItem('chat-input-history', JSON.stringify(next));
  return next;
});
```

### 多行文本时的上下键冲突

原始实现用 `!input.includes('\n')` 判断是否单行，但这会导致切换到多行历史后无法继续向上翻历史。

改为判断**光标位置**：

```tsx
const cursor = el.selectionStart;
const firstNewline = input.indexOf('\n');
const lastNewline = input.lastIndexOf('\n');
const onFirstLine = firstNewline === -1 || cursor <= firstNewline;
const onLastLine = lastNewline === -1 || cursor > lastNewline;

if (event.key === 'ArrowUp' && onFirstLine) {
  // 切换到上一条历史
} else if (event.key === 'ArrowDown' && onLastLine && historyIndex >= 0) {
  // 切换到下一条历史
}
```

这样多行文本内部的光标移动完全不受影响，只有光标在第一行时 `↑` 才触发历史切换，光标在最后一行时 `↓` 才触发。
